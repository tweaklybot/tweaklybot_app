# 📱 Tweakly — Полное руководство по установке и сборке

---

## ЧТО ВЫ ПОЛУЧИЛИ

Полный исходный код Android-приложения **Tweakly** со следующей структурой:

```
Tweakly/
├── app/
│   ├── build.gradle.kts              ← зависимости приложения
│   ├── proguard-rules.pro
│   ├── google-services.json          ← ЗАМЕНИТЬ своим файлом
│   └── src/main/
│       ├── AndroidManifest.xml
│       ├── java/com/example/tweakly/
│       │   ├── MainActivity.kt
│       │   ├── TweaklyApplication.kt
│       │   ├── di/                   ← Hilt DI модули
│       │   ├── data/                 ← Room, Retrofit, Repository
│       │   ├── ui/                   ← Compose экраны
│       │   └── workers/              ← WorkManager
│       └── res/
├── build.gradle.kts
├── settings.gradle.kts
└── gradle/libs.versions.toml
```

## ЧАСТЬ 1: НАСТРОЙКА FIREBASE (ОБЯЗАТЕЛЬНО)

Без Firebase приложение не запустится. Следуйте этим шагам.

### Шаг 1 — Создайте проект Firebase

1. Откройте https://console.firebase.google.com
2. Нажмите **"Создать проект"**
3. Введите название: `Tweakly`
4. Отключите Google Analytics (необязательно) → **"Создать проект"**

### Шаг 2 — Добавьте Android-приложение

1. На главной странице проекта нажмите иконку Android **`</>`**
2. Заполните поля:
   - **Package name:** `com.example.tweakly`
   - **App nickname:** Tweakly
   - **Debug signing certificate SHA-1:** (получите командой ниже)
3. Чтобы получить SHA-1 для debug:
   ```
   Windows:
   keytool -list -v -keystore %USERPROFILE%\.android\debug.keystore -alias androiddebugkey -storepass android -keypass android

   Mac/Linux:
   keytool -list -v -keystore ~/.android/debug.keystore -alias androiddebugkey -storepass android -keypass android
   ```
4. Скопируйте строку `SHA1: XX:XX:XX:...` и вставьте в поле
5. Нажмите **"Зарегистрировать приложение"**

### Шаг 3 — Скачайте google-services.json

1. Нажмите **"Скачать google-services.json"**
2. Замените файл `app/google-services.json` в проекте на скачанный
3. Нажмите **"Далее"** → **"Далее"** → **"Перейти в консоль"**

### Шаг 4 — Включите методы входа

1. В левом меню: **Authentication** → **Sign-in method**
2. Включите **Email/Password**:
   - Нажмите на строку → включите переключатель → **Сохранить**
3. Включите **Google**:
   - Нажмите на строку → включите переключатель
   - Выберите email поддержки проекта → **Сохранить**

### Шаг 5 — Получите Web Client ID для Google Sign-In

1. В Firebase Console: **Authentication** → **Sign-in method** → **Google**
2. Раскройте блок **"Конфигурация SDK для Интернета"**
3. Скопируйте **Web client ID** (формат: `xxxxx.apps.googleusercontent.com`)
4. Откройте файл:
   ```
   app/src/main/java/com/example/tweakly/ui/auth/AuthScreen.kt
   ```
5. Найдите строку:
   ```kotlin
   .requestIdToken("YOUR_WEB_CLIENT_ID")
   ```
6. Замените `YOUR_WEB_CLIENT_ID` на скопированный ID

## ЧАСТЬ 2: УСТАНОВКА ИНСТРУМЕНТОВ

### Вариант A — Android Studio (рекомендуется, проще)

1. Скачайте **Android Studio Hedgehog** или новее:
   https://developer.android.com/studio
2. Установите, запустите, дождитесь загрузки SDK
3. При первом запуске: **SDK Manager** → убедитесь, что установлен **Android SDK 34**

### Вариант B — Command Line (только Gradle + JDK)

1. Установите **JDK 17**:
   - Windows: https://adoptium.net → выберите **Temurin 17**
   - Mac: `brew install openjdk@17`
   - Linux: `sudo apt install openjdk-17-jdk`
2. Проверьте: `java -version` → должно показать `17.x.x`

## ЧАСТЬ 3: ОТКРЫТИЕ ПРОЕКТА В ANDROID STUDIO

1. Запустите Android Studio
2. **File → Open** → выберите папку `Tweakly` (корень проекта, где лежит `settings.gradle.kts`)
3. Дождитесь синхронизации Gradle (внизу будет прогресс-бар, 3–10 минут при первом запуске)
4. Если появились ошибки — см. раздел "Частые проблемы" ниже

## ЧАСТЬ 4: СБОРКА APK

### Способ 1 — Через Android Studio (проще)

1. В меню: **Build → Build Bundle(s) / APK(s) → Build APK(s)**
2. Дождитесь сборки (1–5 минут)
3. Появится уведомление: **"Build successful"** → нажмите **"locate"**
4. APK находится по пути:
   ```
   Tweakly/app/build/outputs/apk/debug/app-debug.apk
   ```

### Способ 2 — Через командную строку

Windows (PowerShell из папки Tweakly):
```powershell
.\gradlew.bat assembleDebug
```

Mac/Linux (Terminal из папки Tweakly):
```bash
chmod +x gradlew
./gradlew assembleDebug
```

После успешной сборки APK будет здесь:
```
app/build/outputs/apk/debug/app-debug.apk
```

## ЧАСТЬ 5: УСТАНОВКА APK НА ТЕЛЕФОН

### На Android-устройство (прямой кабель)

**Подготовка телефона:**
1. **Настройки → О телефоне** → нажмите **"Номер сборки"** 7 раз → появится "Режим разработчика"
2. **Настройки → Для разработчиков** → включите **"Отладка по USB"**
3. Подключите телефон кабелем к компьютеру
4. На телефоне появится запрос — нажмите **"Разрешить"**

**Установка через Android Studio:**
- В верхней панели выберите ваш телефон из выпадающего списка
- Нажмите зелёную кнопку **▶ Run** — приложение установится и откроется

**Установка через adb:**
```bash
adb install app/build/outputs/apk/debug/app-debug.apk
```

### Через файл APK (без кабеля)

1. Скопируйте `app-debug.apk` на телефон (через Telegram, Google Drive, USB)
2. На телефоне откройте файл через файловый менеджер
3. Если появится предупреждение "Установка из неизвестных источников":
   - **Настройки → Безопасность → Установка неизвестных приложений**
   - Разрешите для вашего файлового менеджера / браузера
4. Нажмите **"Установить"**

## ЧАСТЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

### ❌ "Could not resolve com.github.burhanrashid52:PhotoEditor"
**Решение:** Убедитесь, что в `settings.gradle.kts` есть:
```kotlin
maven { url = uri("https://jitpack.io") }
```

### ❌ "Google Sign-In failed" / "DEVELOPER_ERROR"
**Решение:**
- Проверьте, что SHA-1 добавлен в Firebase Console
- Убедитесь, что `google-services.json` заменён на скачанный из Firebase
- Проверьте, что Web Client ID вставлен в `AuthScreen.kt`

### ❌ Gradle sync failed / "SDK location not found"
**Решение:**
- В папке `Tweakly` создайте файл `local.properties`:
  ```
  sdk.dir=C\:\\Users\\YOUR_NAME\\AppData\\Local\\Android\\Sdk
  ```
  (для Windows, замените YOUR_NAME на ваше имя пользователя)
  Mac: `sdk.dir=/Users/YOUR_NAME/Library/Android/sdk`

### ❌ "Unsupported class file major version"
**Решение:** Убедитесь, что используете JDK 17, а не 11 или 21.

### ❌ Room compiler error
**Решение:** Пересоберите проект: **Build → Clean Project** → **Build → Rebuild Project**

### ❌ Приложение крашится при входе
**Решение:** Проверьте интернет-соединение. Firebase Authentication требует сети.

## ДОПОЛНИТЕЛЬНАЯ КОНФИГУРАЦИЯ

### Смена package name
Если хотите сменить `com.example.tweakly` на свой:
1. В Android Studio: правый клик на пакете → **Refactor → Rename**
2. Обновите `applicationId` в `app/build.gradle.kts`
3. Обновите `package_name` в Firebase Console

### Release сборка (для публикации)
```bash
./gradlew assembleRelease
```
Для release нужно создать keystore:
```bash
keytool -genkey -v -keystore tweakly.keystore -alias tweakly -keyalg RSA -keysize 2048 -validity 10000
```
